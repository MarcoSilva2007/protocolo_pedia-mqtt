⚠️ Alerta de Incompatibilidade
Onde NÃO usar o MQTT?

Apesar de ser leve e eficiente, o MQTT não é adequado para todos os cenários. Veja onde ele pode ser uma má escolha:

!

Comunicação Síncrona e Transações Críticas

O MQTT é um protocolo assíncrono baseado em publicação/assinatura. Ele não garante respostas imediatas após o envio de uma mensagem. Por isso, é inadequado para sistemas que exigem confirmação instantânea, como transações bancárias, chamadas de API REST ou operações de CRUD em aplicações web tradicionais.

Nesses casos, protocolos como HTTP/HTTPS são mais seguros e previsíveis.

!

Aplicações Web Tradicionais com Alta Largura de Banda

Em ambientes com dispositivos potentes, conexão estável e sem restrições de energia (como um sistema interno corporativo), o MQTT adiciona complexidade desnecessária.

Configurar um broker, gerenciar tópicos e lidar com QoS pode ser um overhead quando soluções mais simples — como WebSockets ou até mesmo requisições HTTP — já resolvem o problema com menos infraestrutura.
